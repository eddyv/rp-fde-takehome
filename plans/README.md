## Findings considered and rejected

- **Migrate kafka-python → confluent-kafka**: L effort, MED risk on the most
  load-bearing code right before an evaluation. The `compression: none`
  workaround it forced (`README.md:200-202`) is a legitimate talking point
  for the human-written Tradeoffs section instead.
- **CI workflow (GitHub Actions)**: explicitly declined by the maintainer;
  lint/type/test gates stay local (plan 005 documents them).
- **API authentication / bind hardening**: local dev stack serving public
  Wikimedia data; explicit column list folded into plan 003 instead.
- **Failure-routing dedup deferral**: advisor recommended "not now";
  maintainer overruled — became plan 009 with a mandatory verification battery.
- **Direction: pre-LLM short-circuit gate & confidence-tier routing seam**:
  declined — the take-home's post-submission extension will name the real
  requirement; pre-building risks mis-shaping it.
- **Consolidate per-file `make_fixtures` test helpers**: low value; couples
  otherwise-independent suites.
- **config.py guard assert (backoff cap vs poll interval)**: low leverage;
  the consuming arithmetic is already pinned in `test_failures.py`.
- **test-tier `ty` cleanliness (~44 diagnostics from fakes)**: deliberately
  out of scope; check stays scoped to `service/app` (plan 005).
- **No jitter in any backoff formula** (`classifier.BACKOFF_SECONDS`,
  `infra`/`db`'s fixed 2.0s connect-retry delay, `failures.retry_delay_seconds`'s
  deterministic doubling): a real thundering-herd risk if many replicas
  reconnect in lockstep after a shared outage, but this is a production
  design question, not a test-fidelity gap (the tests correctly assert the
  exact deterministic values the code produces) — not planned here; revisit
  if replica count ever grows enough for this to matter in practice.
- **`anthropic.APIConnectionError` branch untested** (`classifier.py:138`
  catches it alongside `RateLimitError`, but no test constructs one): real
  gap, but low risk (simple branch, shares the same `call_model` retry loop
  already covered from the 429/5xx/validation-error angles) — lower value
  than 012's `InterfaceError` gap; not planned this round.
- **`worker.py`/`retrier.py`/`sweeper.py`'s untested `except
  db.CONNECTION_ERRORS: raise` crash branch**: folded into plan 012 rather
  than a separate finding — same root cause (fakes never construct anything
  but `OperationalError`), same fix.

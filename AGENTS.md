## Code Style

Prefer using proper variable names instead of abbreviations. We always favor readability for the humans.

ex: 
w = 5 # bad
width = 5 # good

## Testing

This project is configured with pytest, mutmut, coverage. All changes you make should have a related test for it.
For evaluations, use deepeval.
